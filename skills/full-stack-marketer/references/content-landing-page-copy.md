# Landing Page Copywriting Reference

Complete copy templates, examples, audit checklist, and brief template for the Content &
Page Builder role's copywriting sub-skill.

Source: landing-page-copy/SKILL.md v1.0.0
Research basis: Unbounce research, CXL Institute studies, Copyhackers methodology,
direct response best practices (Eugene Schwartz, Joanna Wiebe).

---

## Awareness Levels (Schwartz Framework)

Target copy to the audience's awareness level:

| Level | Description | Copy Approach |
|---|---|---|
| Unaware | Doesn't know they have the problem | Lead with the problem |
| Problem-aware | Knows the problem, not solutions | Agitate the pain |
| Solution-aware | Knows solutions exist, not yours | Position your approach |
| Product-aware | Knows your product, not convinced | Benefits and proof |
| Most aware | Ready to buy, needs offer details | Price, guarantee, urgency |

## Section-by-Section Copy Templates

### 1. Hero Section (Above the Fold)

**Purpose:** Answer 3 questions within 5 seconds: What is this? Who is it for? Why should I care?

```
[HEADLINE: Primary benefit or transformation in 10 words or fewer]

[SUBHEADLINE: Expand on headline — add specificity, mechanism, or audience. 20-30 words max.]

[CTA BUTTON: Action + Value (e.g., "Start Free Trial" not "Submit")]

[MICROCOPY: Address main anxiety below CTA — "No credit card" / "2-min setup" / "Cancel anytime"]
```

**Headline checklist:**
- Clear primary benefit or transformation
- Specific, not vague ("triple your email opens" not "improve email marketing")
- Matches the ad or email that brought them here (message match)
- 10 words or fewer

**SaaS example:**
```
Headline: "Finally, a CRM salespeople actually use"
Subheadline: "Pipedrive automates busywork so you can focus on closing. Trusted by 100,000+ sales teams."
CTA: "Start Free Trial"
Microcopy: "No credit card • Setup in 5 minutes"
```

**Lead gen example:**
```
Headline: "The 7-Step Checklist That Tripled Our Email Opens"
Subheadline: "Stop guessing. Use the exact framework that grew our list from 0 to 50K."
CTA: "Get the Free Checklist"
Microcopy: "Sent instantly • No spam, unsubscribe anytime"
```

### 2. Problem Section

**Purpose:** Show you understand their pain. Create resonance before presenting the solution.

**Key principles:**
- Use THEIR language (from customer reviews, support tickets, interviews)
- Be specific, not generic
- Agitate but don't manipulate
- 3-5 pain points maximum
- End with a transition to the solution

**Format A — Bullet list:**
```
Sound familiar?
• [Pain point 1 in their words]
• [Pain point 2]
• [Pain point 3]
• [Pain point 4]
```

**Format B — "If you're like most..." Statement:**
```
If you're like most [audience], you've probably...
- Tried [failed solution] and got nowhere
- Spent hours on [tedious task]
- Wondered if [goal] is even possible
```

**Format C — Vivid scenario:**
```
Every Monday, the same thing happens.
[Describe painful scenario in vivid detail]
[Show the consequence]
[Hint at the cost]
```

**Transition:**
"It doesn't have to be this way." or "There's a better way."

### 3. Solution Section

**Purpose:** Introduce your product as the answer to the stated problems.

```
[HEADER: "Enter [Product Name]" or "That's exactly why we built [Product]"]

[DESCRIPTION: 2-3 sentences — what it does, how it solves the stated problems, main differentiator]

[SINGLE KEY DIFFERENTIATOR: The one thing that makes you different]

[VISUAL: Product screenshot or demo GIF]
```

**Key principles:**
- Direct connection to problems stated above
- Focus on outcomes, not features (yet)
- Keep it simple — features come later
- Visual of the product if possible

---

### 4. Features / Benefits Section

**Purpose:** Detail what they get and why each element matters.

**FAB Formula for each feature:** Feature → Advantage → Benefit

```
## Everything you need to [achieve goal]

### [Feature 1: Benefit-first name]
[What it does] → [Why that matters to their life/work]

### [Feature 2: Benefit-first name]
[What it does] → [Why that matters]

### [Feature 3: Benefit-first name]
[What it does] → [Why that matters]
```

**Key principles:**
- Lead with the benefit, support with the feature
- 3-5 core features maximum (don't overwhelm)
- Each feature needs its own icon or visual
- Link features to the pain points stated earlier

**Example:**
```
## Ship faster without breaking things

### One-Click Deploys
Push to production in seconds, not hours. Automatic rollbacks if anything goes wrong.

### Real-Time Error Tracking
Know about issues before your users do. Get alerts with the exact line of code.

### Team Collaboration
Comment on deployments, assign owners, track changes. Everyone stays in sync.
```

### 5. Social Proof Section

**Purpose:** Provide evidence that this works for people like them.

**Types (use 2-3):**
1. **Testimonials** — specific results, photo + name + title
2. **Logos** — recognizable brands, "trusted by" framing, 5-8 logos optimal
3. **Numbers** — users/customers count, results achieved
4. **Ratings/Reviews** — star ratings, third-party review sites
5. **Case studies** — before/after with specific metrics

```
## Join [Number]+ [Audience] who [achieve outcome]

[ROW OF LOGOS OR STATS]

### What our customers say

"[Specific result quote with numbers if possible]"
— [Name], [Title] at [Company]

"[Different angle/benefit highlighted]"
— [Name], [Title] at [Company]
```

**Social proof rules:**
- Use real photos
- Include specific results ("increased revenue 3x" not "great product")
- Match testimonials to target persona
- Avoid generic praise
- Show variety — different use cases and industries

---

### 6. Objection Handling Section

**Purpose:** Address remaining doubts and barriers to conversion.

**Common objections to address:**
- "Is this right for me?"
- "What if it doesn't work?"
- "Is it worth the price?"
- "Is it difficult to use/switch?"
- "Can I trust this company?"

**Format A — FAQ:**
```
## Frequently Asked Questions

**[Question 1]**
[Answer addressing the objection directly]

**[Question 2]**
[Answer]
```

**Format B — Comparison Table:**
| | You (Now) | You (With Product) |
|---|---|---|
| [Aspect 1] | [Current state] | [Better state] |

**Format C — "Is this for me?":**
```
## [Product] is for you if:
✓ You're [characteristic 1]
✓ You want [outcome]
✓ You're ready to [commitment]

## [Product] is NOT for you if:
✗ You're looking for [thing you're not]
✗ You don't [prerequisite]
```

### 7. Final CTA Section

**Purpose:** Recap value and push to conversion.

```
## Ready to [achieve the outcome]?

[1-2 sentences reinforcing the transformation]

• [Benefit 1]
• [Benefit 2]
• [Benefit 3]

[CTA BUTTON]

[Guarantee/risk reversal]
[Final microcopy: "Questions? Contact us"]
```

**Example:**
```
## Ready to stop losing deals to bad data?

Join 10,000+ sales teams who closed more deals with Datafix.

• Real-time contact updates
• 97% accuracy guaranteed
• 2-minute setup, no IT needed

[Start Free—No Credit Card]

30-day money-back guarantee. Cancel anytime.

Questions? Chat with our team →
```

### 8. Footer

**Purpose:** Trust signals and navigation for people who scrolled past everything.

```

[Secondary navigation: Product, Company, Legal]
[Contact / support link]
[Trust signals: certifications, security badges]
[Copyright]
```

## Page Structure by Goal Type

### Lead Gen Page (Free Offer)
1. Hero (clear value of the free offer)
2. What's inside preview
3. Social proof (short)
4. Simple form (email only if possible)
5. Trust signals

**Length:** Short — 1-2 scrolls

### Product Page (SaaS Free Trial)
1. Hero
2. Problem
3. Solution
4. Features (3-5)
5. Social proof
6. Pricing (optional)
7. FAQ
8. Final CTA

**Length:** Medium — 3-4 scrolls

### Sales Page (Paid Product)
1. Hero
2. Problem (expanded)
3. Solution + Story
4. Features/Benefits (detailed)
5. Social proof (extensive)
6. Offer stack
7. Guarantee
8. FAQ (extensive)
9. Final CTA (urgency)

**Length:** Long — 5+ scrolls

### Event / Webinar Page
1. Hero (date/time prominent)
2. What you'll learn
3. Speaker bio
4. Social proof
5. Registration form
6. FAQ (logistics)

**Length:** Short-Medium

## Landing Page Audit Checklist

### Hero (Above the Fold)
- [ ] Headline clear and benefit-driven
- [ ] Subheadline adds specificity
- [ ] CTA visible and action-oriented
- [ ] Value clear within 5 seconds
- [ ] Matches ad/email that drove traffic (message match)

### Message Match
- [ ] Page headline matches traffic source
- [ ] Promise made in ad is delivered on page
- [ ] No bait-and-switch

### Conversion Path
- [ ] Only ONE primary goal
- [ ] CTA repeated throughout (top, middle, bottom)
- [ ] Minimal navigation/exits from the page
- [ ] Form is simple (necessary fields only)

### Trust & Proof
- [ ] Social proof present
- [ ] Testimonials are specific (results with numbers)
- [ ] Logos/badges if relevant
- [ ] Risk reversal clear (guarantee, free trial, cancel anytime)

### Copy Quality
- [ ] Benefits over features
- [ ] "You" language (not "we")
- [ ] Specific numbers and claims
- [ ] Scannable (headers, bullets, short paragraphs)
- [ ] No jargon unless the audience uses it

### Mobile
- [ ] Hero works on mobile without scrolling
- [ ] CTAs are thumb-friendly (large tap targets)
- [ ] Forms easy to complete on mobile
- [ ] Page loads fast (< 3 seconds)
